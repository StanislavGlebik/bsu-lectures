1)Существует текстовый файл содержащий записи вида "Ф.И.О -> телефон". Размер файла неизвестен. Необходимо реализовать программу для поиска телефона по Ф.И.О. При решении данной задачи необходимо реализовать кэш для (Ф.И.О -> телефон) в памяти размером неболее 100 записей и при переполнении кэша предусмотреть следующие настраиваемые стратегии удалении элементов из кэша: по наименьшему количеству обращений, по времени последнего обращения, в произвольном порядке. Программа должна запускаться из командной строки и принимать два параметра: путь к файлу, номер стратегии. Если номер стратегии не введен, то программа выводит весь список доступных стратегий. Далее работа с приложением происходит в интерактивном режиме: на ввод пользователем Ф.И.О программа выдает его телефон. Поиск телефона программой происходит следующим образом: 1)проверяется кэш, если данные найдеты, то пункт 6, иначе пункт 2 2)происходит поиск по файлу, если данные найдены в файле, то пункт 3, иначе пункт 7 3)если кэш переполнен, то пункт 4, иначе пункт 5 4)происходит удаление элемента из кэша, используя выбранную стратегию и переходим к пункту 5 5)телефон записывается в кэш, и переходим к пункту 6 6)данные возвращаются пользователю 7)пользователю выводится сообщение, что телефон не найден. P.S. Не обязательно, но можно 1) использовать предварительную обработку файла для ускорения поиска по файлу. 2) реализовать дополнительные стратегии очистки кэша. 3)Использовать xml файл с произвольным форматом, но сохраняющий структурную идею задачи, т.е. xml файл имеет некоторый уникальный идентификатор пользователя и некоторую дополнительную информацию о нем. 2) Существует csv файла следующего формата: """ column_1, column_2, ..... , column_n some_value_11, some_value_12, ......, some_value_1n .... some_value_m1, some_value_m2, ......, some_value_mn """ Необходимо реализовать программу, которую в интерактивном режиме по запросам от пользователя, следующего формата select column_2, column_1, ... column_k where column_p in/=/like ("column_value_11", "column_value_ps") возвращает все подходящие записи из файла. Программа должна запускаться из командной строки и принимать один параметр: путь к файлу. 3)Необходимо реализовать программу для редактирования csv файлов. Для работы с файлам существуют следующие команды: 1):create <file_name> - позволяет пользователя через запятую ввести название колонок в файле. После нажание кнопки ввода, колонки сохраняются в файла и программа переходит в режим работы с файлом. 2):show <fileName< - выводит на экран содержимое csv файла 3):use fileName - выбирает текущий файл для дальнейшего использования. Команды доступные для работы с файлом и после выбора файла при помлщи команды :use. 1):show columns - выводит имена всех колонок 2):insert (column_1, ... , column_k) values (value_1,... value_k) - добавляет запись в конец csv файла. 3):sort by <column_1, ..., column_p> - сортирует содержимое текущего csv файла, 4):search by -f <column_k> -v <some_value> - выводит на экран все подходящие записи из текущего файла. 5):remove where -f <column_k> -v <some_value> - удаляет из файла все удолетворяющие запросу записи. 4)Написать программу для поиска анаграмм, которые можно составить из введенной пользователем фразы. При реализации программу стоит учитывать следующее: 1)Анаграммы составляются без учета регистра. 2)Анаграммы с одинаковыми словами, но составленных в разной последовательности - являются двумя отличными анаграммами. 3)Программа использует текстовый файл(словарь) для поиска анаграмма. Текстовый файл содержит слова на английском языке.
